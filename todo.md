Тренажёр по математике

Давай сделаем сайт-тренажёр по математике: таблицы умножения, деления, сложения, вычитания.

Требования:
Верстка по БЭМу (блок → элемент → модификатор) https://en.bem.info/methodology/ - чтобы легко поддерживать и развивать. 
Mobile-first - предполагается, что 90% будут открывать его с телефона, чтобы “потупить в очереди”. 
Без использования фреймворков где не нужно, желательно чистый html/css/js.
Минималистичный дизайн в стиле chatgpt/apple.
Светлая и темная темы нативно через @media prefer-color-scheme.
Данные сессии хранятся в локалсторадже.
Сайт можно вынести на экран домой как приложение.
Можно ли из веба вызывать haptic Feedback? Если да, то делать очень мягкую короткую вибрацию при ответе.

Главная страница: 
Segmented Control (одна длинная кнопка, разделенная на 4 сектора) для выбора сложности: 👶🏻 (однозначные числа), 👦🏻 (двузначные),👴🏻 (двузначные + трехзначные), 🧠 (двузначные + трехзначные + четырехзначные). В категории 👴🏻 и 🧠 лучше делать смесь (например, 80% новых сложных примеров и 20% из предыдущего уровня)

Кнопки для выбора режима игры - сетка в 2 столбца с кнопками-иконками: сложение, вычитание, умножение, деление.
Под ними широкая кнопка ”Таблица умножения” и ниже широкая “Всё подряд” - при нажатии на нее выпадают примеры из всех категорий.

Страница игры:
Прогресс в виде визуального прогресс-бара [1/12] (12 заданий в одном раунде)Крупно вопрос-пример 2 × 4 
4 кнопки с вариантами ответов, всегда перемешаны, один верный ответ. При нажатии на верный ответ вся кнопка загорается зеленым и через 200 мс переключается вопрос. При нажатии на неверный ответ кнопка загорается красным, а верный ответ зеленым, переключается на новый пример только после тапа пользователя по любой кнопке.

Страница результата:
(Эффект Конфетти на всю страницу при результате 12/12)
Заголовок “Результат”
[n/12]
Кнопка “Разобрать ошибки” - открывается режим игры, в котором отображаются только примеры с неверными ответами пользователя за последнюю сессию, если пользователь снова ошибается в этом режиме, пример должен возвращаться в очередь, пока не будет решен верно.
Кнопка “Еще раунд”
Кнопки с главной страницы с выбором режима игры (умножение, деление и т.д.) - чтобы сразу начать новую игру.

Алгоритм генерации задач
Цель: Генерировать объект задачи с примером, правильным ответом и тремя правдоподобными вариантами ошибок (дистракторами).
1. Параметры сложности:
* 👶🏻 (Низкая): Числа 1–10 для всех операций.
* 👦🏻 (Средняя): $+/-$ (10–99); $\times/\div$ (множители 2–19 и 2–9).
* 👴🏻 (Высокая): $+/-$ (100–999); $\times/\div$ (множители 10–50 и 2–15).
* 🧠 (Экстремальная): $+/-$ (1000–9999); $\times/\div$ (множители 10–99 и 10–50).
2. Алгоритмы генерации:
* Вычитание: Результат всегда $\ge 0$ (вычитать меньшее из большего).
* Деление: Генерировать через умножение ($B \times C = A$), где игроку дается $A / B$, чтобы результат всегда был целым числом.
* Умножение: Использовать "асимметричные" множители на средних уровнях (двузначное на однозначное), чтобы расчет оставался возможным в уме.
3. Логика вариантов ответа (4 кнопки):
* 1 верный ответ.
* 3 дистрактора: 1. Смещение на $\pm 1$ или $\pm 10$ от верного ответа.2. Ошибка "перестановки" (например, 42 вместо 24), если уместно.3. Случайное число в диапазоне $\pm 20\%$ от ответа, имеющее ту же последнюю цифру, что и правильный результат.
4. Формат вывода (JSON):
JSON

{
  "question": "45 × 7",
  "answer": 315,
  "options": [315, 305, 314, 325] 
}
(Массив options должен быть перемешан перед выдачей).
